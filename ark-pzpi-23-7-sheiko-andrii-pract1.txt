2 ЗАВДАННЯ
	
Студент повинен обрати мову програмування, дослідити основні рекомендації для написання якісного коду у цій мові, продемонструвати ці рекомендації на прикладах коду, обґрунтувати кожну рекомендацію і надати детальні пояснення. 
Студенту необхідно обрати мову програмування для аналізу. Після вибору мови програмування студенту потрібно ознайомитися з основними рекомендаціями для написання якісного та чистого коду у цій мові. Це можуть бути стильові рекомендації, правила найменування змінних, функцій та класів, структура коду, принципи рефакторингу, оптимізація продуктивності, обробка помилок, дотримання парадигм програмування, тестування та документування коду тощо.
Студент повинен підготувати один або кілька прикладів коду, які демонструють, як потрібно застосовувати наведені рекомендації. Приклад коду може бути загальним і об'єднувати декілька рекомендацій, або ж студент може підготувати кілька окремих прикладів для різних аспектів рекомендацій.
Студент повинен надати докладні пояснення щодо кожної рекомендації, підготувати слайди або текстову доповідь для презентації власного аналізу та представити свою доповідь викладачу.











3 ОПИС ВИКОНАНОЇ РОБОТИ

3.1 Вибір мови програмування
	Для аналізу конвенцій кодування було обрано мову програмування C. C є однією з фундаментальних мов програмуваннся, яка широко використовується у системному програмуванні, розробці операційних систем, вбудованих систем та інших галузях, де критично важливі ефективність та низькорівневий контроль. Високий рівень свободи у мові C робить стандартизацію коду особливо важливою для забезпечення читабельності, переносимості та легкості супроводу.

3.2 Важливість конвенцій кодування
	Дотримання конвенцій кодування є критичним для життєвого циклу програмного забезпечення, особливо у великих проектах, таких як операційні системи або системні бібліотеки. Код, написаний згідно із загальноприйнятими стандартами, значно полегшує командну роботу, читання, супровід та виправлення помилок. «GNU Coding Standards (GCS)» були розроблені саме для забезпечення високої якості та узгодженості коду в межах проекту GNU, але їх принципи універсальні та можуть бути застосовані до будь-якого проекту на C

3.3 Джерела конвенцій
	Основним джерелом конвенцій для даної роботи є офіційний документ «GNU Coding Standards», розроблений Фондом вільного програмного забезпечення (FSF). Цей стандарт охоплює широке коло питань: від іменування змінних до структури проектів та міжплатформної переносимості. GCS є стандартом для багатьох проектів з відкритим вихідним кодом, що робить його вивчення обов'язковим для сучасного розробника.

3.4 Правила іменування
	GCS рекомендує використовувати змістовні та описові імена для всіх ідентифікаторів. Стиль іменування – lowercase_with_underscores. Це забезпечує читабельність та узгодженість коду.
1   // Погано
2  int a, tmp;
3  void fnd();
4  void proc_data();

1  // Добре
2  int number_of_users;
3  char* file_name;
4  void find_user_by_id();
5  void process_incoming_data();


3.5 Відступи та пробіли
	Для відступів використовуються 2 пробіли, не табуляція. Пробіли навколо операторів та після ком у списках покращують читабельність.
1  // Погано
2  for(int i=0;i<10;i++){...}
3
4  // Добре
5  for (int i = 0; i < 10; i++) {
6    // ...
7  }


3.6 Розміщення фігурних дужок (Стиль K&R)
	GCS приймає стиль K&R (Kernighan & Ritchie): відкриваюча дужка { розміщується в кінці рядка оператора, закриваюча } – на новому рядку на тому ж рівні відступу.
1  // Добре
2  if (condition) {
3    // ...
4  } else {
5    // ...
6  }

3.7 Довжина рядка та розбиття коду
	Рекомендована максимальна довжина рядка - 79 символів. Це забезпечує коректне відображення в терміналах, інструментах порівняння версій (diff) та під час код-рев'ю.
1  // Добре
2  long_variable_name = some_complex_function(argument_one,
3                                             argument_two,
4                                             argument_three);


3.8 Коментарі та документація
	Коментарі повинні пояснювати чому код працює так, а не що він робить. Для коментування використовуються «//» для коротких коментарів та «/* ... */» для довгих. Функції мають супроводжуватися коментарями, що описують їх призначення, аргументи та значення, що повертається.
1  /*
2   * calculate_square - обчислює квадрат числа
3   * x - вхідне число
4   *
5   * повертає квадрат числа x
6   */
7  int calculate_square(int x) {
8    return x * x; //просте піднесення до квадрату
9  }

3.9 Структура файлів та Include Guard
	Заголовочні файли (.h) містять оголошення, реалізація знаходиться у файлах (.c). Кожен заголовочний файл повинен мати Include Guard для запобігання множинного включення.
1  // example.h
2  #ifndef EXAMPLE_H
3  #define EXAMPLE_H
4  
5  #include <stdio.h>
6  #include "local_header.h"
7  
8  void public_function(int parameter);
9  
10  #endif /* EXAMPLE_H */

3.10 Обробка помилок
	Функції повинні перевіряти вхідні дані на коректність. Як правило, успіх позначається значенням 0, а помилка – ненульовим або від’ємним значенням. Для стандартизації кодів помилок використовується errno.
1  #include <errno.h>
2  
3  int safe_divide(int a, int b, int *result) {
4    if (b == 0) {
5      errno = EINVAL; // недійсний аргумент
6      return -1;
7    }
8    *result = a / b;
9    return 0; // успіх
10  }

3.11 Приклади рефакторингу коду
	Наведемо приклад коду до та після рефакторингу, що демонструє застосування вищезазначених правил..
1  // До рефакторингу (поганий стиль)
2  #include <stdio.h>
3  int main(){int 4x=5,y=10,z;z=x+y;if(z>10){printf("Big");}else{printf("Small");}return0;}
5  
6  // Після рефакторингу (згідно з GCS)
7  #include <stdio.h>
8  
9  int main() {
10    int first_number = 5;
11    int second_number = 10;
12    int sum = first_number + second_number;
13  
14    if (sum > 10) {
15      printf("The sum is big.\n");
16    } else {
17      printf("The sum is small.\n");
18    }
19  
20    return 0;
21  }
























ВИСНОВКИ

У результаті виконання практичного заняття було вивчено основні рекомендації для написання коду мовою програмування C, описані у стандарті GNU Coding Standards. 
Було розглянуто ключові аспекти якісного коду: змістовне іменування, послідовне форматування, використання коментарів, структуру файлів та належну обробку помилок. На основі цього було створено презентацію, яка наочно демонструє важливість дотримання стандартів та їх практичне застосування. 
Виконана робота підтвердила, що дотримання конвенцій кодування є невід'ємною частиною професійної розробки програмного забезпечення, що забезпечує створення стабільного, читабельного та легко супроводжуваного коду.
